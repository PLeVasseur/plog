Module zero_copy

Zero-copy communication-layer facade.

This module exposes the L2 operations that preserve zero-copy semantics: stable/no-zero publish and subscription with the transport’s receive lease type delivered directly to the listener. The facade does not adapt or copy received payloads, so selected-wire transports retain their typed UWireRx decode surface.

RPC server and request handling remain at the UZeroCopyTransport layer because their non-copying shape is tied to the transport receive lease type. Any future convenience that copies out of a receive lease must use a copying name and must not claim no-copy behavior.
